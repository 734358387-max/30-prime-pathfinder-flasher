# 30-prime-pathfinder-flasher
一键烧录与自动唤醒工具 v8.3 (安全防作弊版)

## 功能介绍 (Features)
这是基于逆向工程原始 USB 通讯协议开发的自定义烧录软件。v8.3 版本引入了多项关键的安全防作弊机制、云端固件自动拉取以及真正的“一键全自动硬件唤醒”等重磅功能，以保证生产环境下烧录过程的可靠性、安全性与便捷性。

主要亮点功能包括：
1. **自动硬件唤醒 (Auto Wakeup)**：通过 Python (pywinusb.hid) 重现了自定义开发板的 USB 唤醒指令 (`0xff, 0xff, 0x02, 0xf5, 0xf7...`)，无需繁琐的物理按键操作，一键即可让设备从隐藏状态进入 U盘挂载模式。
2. **云端固件直连与 MD5 安全校验**：支持从阿里云 OSS 云端直接拉取 `index.txt` 并下载对应的固件（`.zip` 包）。在解压烧录前会严格进行 MD5 校验，避免网络波动导致的固件损坏刷入报错故障，实现“防作弊”安全管控。
3. **混合资料源选择 (Hybrid Source)**：保留本地文件夹直接批量烧录功能，适配多种调试与生产场景，灵活切换。
4. **稳定容错的烧录机制**：全自动清空旧固件并批量写入新固件，针对特殊文件系统的权限报错做了优雅的回退兼容处理 (Fallback copy)。

---

## 软件实现方法 (Implementation Details)
- **开发语言与 GUI**：使用 Python 3 和 Tkinter 开发图形界面，运用多线程（`threading`）确保在执行网络下载、硬件轮询或大文件复制时不阻塞主线程 UI。
- **通信逆向与识别**：软件内置针对三种指定 PID/VID 组合 (`28E9:028F`, `0483:5750`, `28E9:0285`) 的 HID 设备识别机制。
- **系统底层控制**：使用 `ctypes.windll` 深度调用 Windows 底层 API（如 `GetLogicalDrives`、`GetVolumeInformationW`、`SendMessageTimeoutW`），以实现在不插拔 USB 的情况下强制刷新系统设备管理器 (PnP Device Rescan) 识别目标 U 盘。

## 操作手册 (Operation Manual)

在使用 v8.3 烧录软件前，请确保开发板已通过 USB 连接到电脑。

### 第一步：准备硬件
点击界面上方的 **“点击：自动唤醒并激活U盘模式”** 按钮。
此时软件会在后台自动向目标板发送特定的通信报文（HID Report），并在成功通讯后强制要求操作系统重新挂载 U 盘驱动器。唤醒结束后，在界面提示框内会出现“✅ 唤醒且系统识别成功”。

### 第二步：选择资料源
您可以根据当前任务选择两种不同的固件提供方式：
- **云端固件库（推荐）**：选择后，软件会自动连接到远程固件仓获取可用固件列表及其 MD5 值。在下拉框内选中目标固件（例如特定的生产发版包），系统负责自动下载与一致性校验。
- **本地文件夹**：如果您在进行线下固件开发测试，可选择此模式，点击“浏览…”选择存有编译输出文件的本地目录。

### 第三步：选择目标 U盘
请在下拉框中确认并选择刚刚由设备唤醒出来的目标 U盘盘符。
如果您刚才错过了唤醒响应时间，可以通过点击“刷新列表”来手动识别设备。

### 第四步：一键烧录
点击底部红色的 **“一键烧录”** 大按钮。
软件会自动执行：
1.（如选云端则下载并验证安全指纹）
2. 清空选定 U 盘上的原始数据
3. 复制新固件文件至目标设备并校验进度
进度条达到 100% 后，弹出烧录成功提示框，此时您可以安全拔出设备。
